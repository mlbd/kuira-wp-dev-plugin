# Acme Notes — reference output of `/wp-new`

This is **not a real plugin to ship.** It's a complete, WPCS-compliant plugin
generated by the kuira `/wp-new` wizard, committed to the toolkit repo so you can see
exactly what the toolkit produces — and so CI can hold that output to the same
standards the toolkit enforces.

## The configuration that generated it

| Wizard question | Answer |
|-----------------|--------|
| Frontend stack | **React** (`@wordpress/scripts`) |
| Distribution | **Both** (wordpress.org-ready) |
| Gutenberg block | No |
| WooCommerce | No |
| Features | Settings page · REST/AJAX · Custom DB table |
| Quality tooling | Testing (wp-env + PHPUnit) · Static analysis (PHPStan) |
| Ship & integrate | (none) |
| Git & commits | Manual |

## What to look at

| Concern | File | Demonstrates |
|---------|------|--------------|
| Bootstrap | `acme-notes.php`, `includes/class-acme-notes.php` | Header, constants, singleton, migration check |
| Secure REST | `includes/class-acme-notes-rest-controller.php` | `permission_callback` (capability), sanitized args, `WP_Error` statuses |
| Custom table | `includes/class-acme-notes-db.php` | `dbDelta` schema, `$wpdb->prepare`, version-gated install |
| React admin | `includes/class-acme-notes-admin.php`, `src/` | Scoped enqueue, `index.asset.php`, `X-WP-Nonce` REST calls |
| Tests | `tests/notes-test.php` | Table-exists, CRUD, and a 403-on-insufficient-capability test |
| Playground | `blueprint.json` | Docker-free preview/demo |
| Claude setup | `CLAUDE.md`, `.claude/settings.json` | The project orientation `/wp-new` writes |

## Run it

```bash
composer install
npm install && npm run build
npm run wp-env start          # http://localhost:8888  (admin / password)
composer test                 # PHPUnit
composer analyze              # PHPStan
```

Or preview without Docker:

```bash
npx @wp-playground/cli@latest server \
  --mount=.:/wordpress/wp-content/plugins/acme-notes \
  --blueprint=blueprint.json
```
