# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres
to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- `wp-new` skill — interactive `/wp-new` wizard. Runs a multi-step onboarding
  (frontend stack, wordpress.org vs self-hosted, Gutenberg blocks, WooCommerce,
  features, dev tooling, and git-commit behavior), takes a free-text plugin
  description, confirms a plan, then generates a tailored plugin file structure
  (composing the focused skills) plus the Claude Code project setup (`CLAUDE.md` +
  `.claude/settings.json`). The commit choice is honored: **manual** (default,
  Claude never commits unasked), **auto-commit** (adds a Stop hook), or **never**
  (no git init; `git commit`/`push` denied).
- `wp-scaffold` skill — generates a complete, WPCS-compliant new-plugin skeleton
  (main file + header, singleton bootstrap class, activator/deactivator, an admin
  page that enqueues assets, `composer.json` wired to WPCS, `phpcs.xml.dist`, optional
  WooCommerce/HPOS).
- `wp-scaffold` frontend-stack selection — wires up a full build pipeline for the
  developer's choice of **React** (`@wordpress/scripts`), **Vue** (Vite), or **vanilla**
  (jQuery + CSS), including the admin mount point and a `wp_localize_script` REST
  URL + nonce payload so the API is callable from the first line of JS.
- `wp-readme` skill — generates/validates a spec-compliant WordPress.org `readme.txt`
  (header block, short description, sections, changelog, upgrade notice; checks
  Stable tag ↔ version, ≤5 tags, required headers).
- Six opt-in lifecycle skills (run only when explicitly asked, confirm before
  installing dependencies):
  - `wp-endpoint` — secure-by-default REST/AJAX handler scaffolder (permission_callback,
    nonce, capability, per-arg sanitization).
  - `wp-block` — Gutenberg block scaffolder (`block.json` apiVersion 3, static/dynamic).
  - `wp-db` — `dbDelta` schema creation, version-gated migrations, prepared CRUD, uninstall.
  - `wp-test` — `@wordpress/env` local WP + PHPUnit WordPress test suite, with REST/AJAX tests.
  - `wp-i18n` — text-domain verification, string wrapping, `.pot` generation, JS translations.
  - `wp-plugin-check` — runs the official Plugin Check (PCP) against wordpress.org guidelines.
- `CONTRIBUTING.md`, `CHANGELOG.md`, and GitHub issue/PR templates for the public
  open-source release.
- Ship-pipeline skills (opt-in): `wp-analyze` (PHPStan static analysis + WP stubs),
  `wp-e2e` (Playwright browser tests against wp-env), `wp-deploy` (GitHub Actions CI
  + wordpress.org SVN deploy on tag via the 10up action).
- Maintenance skills (opt-in): `wp-modernize` (incremental legacy→modern refactor),
  `wp-php8` (PHPCompatibilityWP scan + fixes), `wp-debug` (WP_DEBUG + debug.log
  diagnosis), `wp-hook-docs` (generate a `HOOKS.md` reference).
- `wp-help` skill (`/wp-help`) — read-only command map of every skill, agent, and hook.
- Two auditor agents: `wp-performance-auditor` (queries-in-loops, autoloaded-option
  bloat, missing caching) and `wp-a11y-auditor` (admin UI accessibility, WCAG 2.1 AA).
- Platform hooks (low-noise): `SessionStart` setup reminders (silent unless deps are
  missing); a fast `php -l` syntax check before WPCS on save; an **opt-in** commit
  quality gate (`KUIRA_COMMIT_GATE=1`) that blocks `git commit` on staged PHP with
  syntax/WPCS errors.
- Optional `statusline.sh` — WP-aware statusline (plugin name + version); not
  auto-enabled.
- `/wp-new` wizard extended with the new opt-in choices (Playwright E2E, PHPStan,
  GitHub Actions CI, wordpress.org auto-deploy, live-WordPress MCP).
- `wp-security-audit` now also runs `composer audit` / `npm audit` for dependency
  vulnerabilities.

- `wp-playground` skill — instant WASM WordPress via the Playground CLI
  (`@wp-playground/cli`; `wp-now` is deprecated) + a generated Blueprint for
  Docker-free demos, manual QA, and CI smoke tests.
- Repo self-CI: `scripts/validate.sh` + `.github/workflows/validate.yml` —
  validates all JSON, skill/agent name↔path, required frontmatter, and shellchecks
  the hook scripts.
- OSS hygiene: `SECURITY.md`, `CODE_OF_CONDUCT.md`, and `docs/AUTHORING.md`
  (skill/agent/hook authoring guide).
- Enriched `plugin.json` / `marketplace.json` metadata (homepage, repository, bugs,
  keywords, category) and expanded the `settings.json` permission allowlist for the
  new tooling (phpcs/phpcbf/phpstan/phpunit/node/vendor-bin).

- `examples/acme-notes` — a complete reference plugin generated by `/wp-new` (React +
  secure REST + custom table + PHPUnit tests + Playground Blueprint + Claude Code
  setup), committed so the toolkit's output is visible and CI-checked.
- `fixtures/` — `vulnerable/` (seven planted security issues, `[V1]`–`[V7]`) and
  `clean/` (the fixes), so the security auditors can be *proven* effective, plus a
  `fixtures/README.md` mapping each issue to its expected detection.

### Changed
- **Licensed under GPL-2.0-or-later** (added the `LICENSE` file; switched all four
  `plugin.json` manifests, the README badge/footer, and CONTRIBUTING from MIT). The
  toolkit and the WordPress plugins it generates now share one license — the standard
  for the WordPress.org directory.
- **Restructured into a modular marketplace of four plugins** under `plugins/`:
  **kuira-wp-core** (build essentials + safety hooks, MCP, statusline — 8 skills, 1
  agent), **kuira-wp-quality** (test/analyze/audit — 6 skills, 4 agents, opt-in
  commit gate), **kuira-wp-ship** (release/deploy/docs — 5 skills), and
  **kuira-wp-maintain** (modernize/php8/debug — 3 skills). Install only the lifecycle
  stages you need. `marketplace.json` now lists all four; each has its own
  `plugin.json`. The `/wp-new` wizard degrades gracefully when a referenced companion
  module isn't installed. `scripts/validate.sh` updated to validate the per-module
  layout (incl. plugin-manifest name↔folder checks).
- Stop notification hook gained a Windows (BurntToast) branch — previously no-op on
  Windows.
- **`wp-security-audit` + `wp-security-auditor`: fixed a real reflected-XSS detection
  gap** — the previous `echo \$` pattern missed output built by concatenation
  (`echo 'x' . $_POST[...]`). Both now also scan for request superglobals reaching
  `echo`/`print` anywhere on the line. (Found by the new fixtures.)
- `wp-scaffold` notes the 2026 `@wordpress/scripts` → `@wordpress/build` engine
  transition (no action needed; `wp-scripts` commands unchanged).
- Rewrote `README.md` around the `/plugin marketplace add` install flow and the
  scaffold-first workflow.
- Genericized `wp-context` and `wp-ui-visual` — removed hardcoded private product
  names, text domains, and slugs in favor of neutral placeholders.

### Removed
- `oasis-workflow` skill and all references to specific commercial plugins, so the
  release contains no proprietary/internal content.

### Fixed
- `bash-guard.sh` `rm -rf` guard: replaced an unsupported `grep -E` negative
  lookahead (which silently never matched) with a portable match + `/tmp` allow-list.
